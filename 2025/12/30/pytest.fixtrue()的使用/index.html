2、Fixture的作用范围

Fixture 的作用范围可以通过 scope 参数来控制。scope 参数可以取以下值：

  • function: 每个测试函数都会调用一次（默认值）。
  • class: 每个测试类调用一次。
  • module: 每个模块调用一次。
  • session: 每个会话调用一次

function 作用范围：适用于独立的测试用例，避免跨用例数据污染，常用于简单的状态设置或资源管理。

class 作用范围：适用于一组共享资源的测试用例，例如需要在同一个数据库连接下运行的多个查询测试。class 级别的 fixture 在每个测试类中只初始化一次

module 作用范围：适用于整个模块内的测试共用的资源，如一次性建立的数据库连接或缓存服务。模块级别的 fixture 有助于减少重复的 setup 操作，优化执行效率。

session 作用范围：适用于整个测试会话的全局资源，例如配置文件加载、测试数据库创建等，适合需要频繁调用的共享资源。

输出结果：

选择 scope 的核心原则是：在保证测试间必要隔离的前提下，尽可能使用更大的 scope 来提升性能。

  • 追求独立性：用 function。
  • 类内共享：用 class。
  • 文件内共享：用 module。
  • 全局共享：用 session。

根据这个原则，一个典型的测试项目可能是：大部分基础数据用 function 级别保证独立；数据库连接用 module 或 session 级别复用；而全局的配置或驱动则用 session 级别只初始化一次。

二、共享fixture 函数：conftest.py

conftest.py文件的注意事项

  • conftest.py文件名是固定的，不可更改。
  • 可以在不同的.py文件中使用同一个fixture函数。
  • conftest.py文件需要和运行的用例放在同一目录中，并且该目录中需要有*init.py*文件。
  • 如果希望fixture共享给所有测试，可以把conftest.py文件放在测试框架的根目录下。
